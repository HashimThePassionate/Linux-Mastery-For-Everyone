# ☁️ The Shift to Cloud Computing

Recent years have witnessed a dramatic transformation in the technology landscape, moving from **on-premises computing platforms** to **private and public clouds**. In today's fast-paced world, businesses and organizations face a critical need: to deploy and run applications on infrastructure that is highly scalable, efficient, and secure.

Maintaining this level of security and performance using traditional, on-premises resources requires immense expertise and financial investment. In comparison to modern public cloud offerings, the cost of running your own data centers is becoming increasingly difficult to justify. While large enterprises move slowly, businesses of all sizes are adopting public cloud services in record numbers.

---

## 🚰 The "Services on Tap" Metaphor

One of the most effective ways to understand cloud computing is through the metaphor of **"Application Services on Tap."**

Just like water from a faucet, resources are available instantly when you need them:

* **Need more resources?** Turn on the tap. You can provision virtual machines or instances in any quantity required. This is known as **Horizontal Scaling**.
* **Need more power?** You can increase the CPU or memory for specific instances. This is known as **Vertical Scaling**.
* **Finished with the resources?** Simply turn off the tap.

**Public Cloud Services** provide these capabilities at relatively low rates. This eliminates the massive operational overhead associated with maintaining physical infrastructure that could support such dynamic features.

## 🎯 Section Goals: AWS and Azure

This Section focuses on the two giants of the public cloud industry: **Amazon Web Services (AWS)** and **Microsoft Azure**.

We will provide practical guidance on deploying applications in the cloud, specifically focusing on typical cloud management workloads using two primary tools:

1. **The Web Administration Console:** Graphical user interface for management.
2. **The Command-Line Interface (CLI):** Text-based tool for automation and efficiency.

### 🎓 Learning Outcomes

By the end of this guide, you will master the following skills for both AWS and Azure:

* **Web Console Management:** How to navigate and use the visual dashboards.
* **CLI Mastery:** How to use the AWS CLI and Azure CLI to manage resources programmatically.
* **Decision Making:** How to make prudent choices when creating resources, striking a sensible balance between **Performance** and **Cost**.
